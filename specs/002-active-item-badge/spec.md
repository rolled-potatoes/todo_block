# Feature Specification: 진행중 아이템 확장프로그램 아이콘 배지

**Feature Branch**: `002-active-item-badge`  
**Created**: 2026-03-07  
**Status**: Draft  
**Input**: User description: "현재 진행중인 아이템이 있을때 확장프로그램 아이콘에 표시가 되어서 인지할 수 있으면 좋겠다."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 진행중 아이템 배지 표시 (Priority: P1)

사용자가 할 일 목록에서 하나의 아이템을 "진행중(in_progress)" 상태로 전환하면, 브라우저 툴바에 있는 확장프로그램 아이콘에 시각적 배지가 표시된다. 이를 통해 사용자는 팝업을 열지 않고도 현재 집중 모드가 활성화되어 있음을 즉시 인지할 수 있다.

**Why this priority**: 이 기능의 핵심 가치는 "팝업을 열지 않고도 진행중인 작업이 있는지 한눈에 파악"하는 것이다. 배지가 표시되지 않으면 사용자는 매번 팝업을 클릭해서 확인해야 하므로, 이 스토리가 가장 높은 우선순위를 갖는다.

**Independent Test**: 할 일 아이템 하나를 진행중으로 전환한 뒤, 브라우저 툴바의 확장프로그램 아이콘에 배지가 나타나는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 모든 할 일이 idle 또는 completed 상태, **When** 사용자가 하나의 할 일을 in_progress로 전환, **Then** 확장프로그램 아이콘에 시각적으로 구별 가능한 배지가 표시된다
2. **Given** 하나의 할 일이 in_progress 상태, **When** 사용자가 해당 할 일을 completed 또는 idle로 전환, **Then** 확장프로그램 아이콘의 배지가 즉시 사라진다
3. **Given** 하나의 할 일이 in_progress 상태, **When** 사용자가 브라우저 툴바를 확인, **Then** 배지를 통해 현재 집중 모드가 활성화되어 있음을 팝업을 열지 않고 인지할 수 있다

---

### User Story 2 - 브라우저 재시작 시 배지 복원 (Priority: P2)

사용자가 브라우저를 종료했다가 다시 열었을 때, 이전에 진행중이던 아이템이 있다면 배지가 자동으로 복원된다. 기존에 차단 규칙(DNR rules)이 복원되는 것과 동일한 시점에 배지도 함께 복원되어야 한다.

**Why this priority**: 브라우저 재시작은 빈번하게 발생하며, 배지가 복원되지 않으면 사용자가 집중 모드가 활성 상태인지 인지하지 못할 수 있다. P1의 핵심 가치를 지속적으로 유지하기 위해 필수적이다.

**Independent Test**: 진행중인 아이템이 있는 상태에서 브라우저를 종료 후 재시작하고, 확장프로그램 아이콘에 배지가 복원되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 하나의 할 일이 in_progress 상태에서 브라우저가 종료됨, **When** 브라우저를 다시 시작, **Then** 확장프로그램 아이콘에 배지가 자동으로 복원된다
2. **Given** 모든 할 일이 idle 또는 completed 상태에서 브라우저가 종료됨, **When** 브라우저를 다시 시작, **Then** 확장프로그램 아이콘에 배지가 표시되지 않는다

---

### User Story 3 - 배지로 진행중인 작업명 확인 (Priority: P3)

사용자가 확장프로그램 아이콘 위에 마우스를 올리면 (hover/tooltip), 현재 진행중인 할 일의 제목을 확인할 수 있다. 이를 통해 팝업을 열지 않고도 어떤 작업에 집중하고 있는지 알 수 있다.

**Why this priority**: P1과 P2가 "진행중인 작업이 있는지" 여부를 알려주는 반면, 이 스토리는 "어떤 작업인지"까지 알려주는 추가적인 편의 기능이다. 필수는 아니지만 사용자 경험을 크게 향상시킨다.

**Independent Test**: 진행중인 아이템이 있는 상태에서 확장프로그램 아이콘에 마우스를 올려 툴팁에 작업명이 표시되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** "보고서 작성" 이라는 할 일이 in_progress 상태, **When** 사용자가 확장프로그램 아이콘에 마우스를 올림, **Then** 툴팁에 "보고서 작성" 이라는 현재 진행중인 작업명이 표시된다
2. **Given** 진행중인 아이템이 없는 상태, **When** 사용자가 확장프로그램 아이콘에 마우스를 올림, **Then** 기본 확장프로그램 이름(또는 기본 설명)이 툴팁에 표시된다

---

### Edge Cases

- 확장프로그램이 설치된 직후(아직 API 토큰이 설정되지 않은 상태)에는 배지가 표시되지 않아야 한다
- 네트워크 오류 등으로 할 일 목록을 불러올 수 없는 상태에서도, 로컬에 저장된 진행중 아이템 정보에 기반하여 배지가 올바르게 유지되어야 한다
- 사용자가 Todoist에서 직접(웹/앱) 진행중이던 할 일을 완료한 경우, 다음 동기화 시 배지가 적절히 업데이트되어야 한다
- 확장프로그램 업데이트 시 진행중 상태의 아이템이 있다면 배지가 유지되어야 한다

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 할 일의 상태가 in_progress로 변경될 때 확장프로그램 아이콘에 시각적 배지를 즉시 표시해야 한다 (MUST)
- **FR-002**: 시스템은 진행중인 할 일이 없어질 때(completed 또는 idle로 전환) 배지를 즉시 제거해야 한다 (MUST)
- **FR-003**: 배지는 일반 아이콘 상태와 명확히 구별 가능해야 한다 (MUST) — 색상, 텍스트, 또는 아이콘 변경 등의 방법으로 사용자가 즉시 인지할 수 있어야 한다
- **FR-004**: 시스템은 브라우저 시작 시 저장된 진행중 아이템 상태를 확인하고, 해당 아이템이 있으면 배지를 자동 복원해야 한다 (MUST)
- **FR-005**: 시스템은 확장프로그램 설치/업데이트 시 저장된 진행중 아이템 상태를 확인하고, 해당 아이템이 있으면 배지를 자동 복원해야 한다 (MUST)
- **FR-006**: 시스템은 진행중인 할 일이 있을 때 확장프로그램 아이콘의 툴팁에 해당 할 일의 제목을 표시해야 한다 (SHOULD)
- **FR-007**: 시스템은 진행중인 할 일이 없을 때 확장프로그램 아이콘의 툴팁을 기본값으로 복원해야 한다 (SHOULD)

## Assumptions

- 기존 시스템에서 이미 in_progress 상태인 할 일의 ID를 로컬 스토리지에 저장하고 있으므로, 배지 표시 여부를 결정하기 위한 별도의 데이터 저장은 불필요하다
- 한 번에 하나의 할 일만 in_progress 상태가 될 수 있다는 기존 제약 조건이 유지된다 (따라서 배지에 개수 표시는 불필요)
- 배지의 시각적 표현은 간결해야 한다 — 단순한 색상 점(dot)이나 짧은 텍스트(예: "ON" 또는 "●") 형태가 적합하다
- 배지 표시/제거는 상태 변경 이벤트와 동기적으로 처리되며, 별도의 폴링이나 주기적 확인은 필요하지 않다

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 할 일을 in_progress로 전환한 후 1초 이내에 확장프로그램 아이콘에 배지가 표시된다
- **SC-002**: 사용자가 진행중인 할 일을 완료 또는 일시정지한 후 1초 이내에 배지가 사라진다
- **SC-003**: 브라우저 재시작 후 확장프로그램이 로드되는 즉시 배지가 복원되어, 사용자가 첫 번째 탭을 열기 전에 집중 모드 상태를 인지할 수 있다
- **SC-004**: 사용자가 팝업을 열지 않고도 집중 모드 활성화 여부를 100% 정확하게 인지할 수 있다
