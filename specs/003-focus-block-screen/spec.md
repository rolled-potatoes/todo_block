# Feature Specification: 집중 차단 화면 (Focus Block Screen)

**Feature Branch**: `003-focus-block-screen`  
**Created**: 2026-03-07  
**Status**: Draft  
**Input**: User description: "차단되는 화면에서 현재 업무를 수행중이라서 집중하라는 경고 문구가 뜨는 창으로 구성했으면 좋겠다. 지금은 아니지만 추후에는 사용자가 커스텀해서 업로드하여 구성하면 재미있을것 같어."

## Clarifications

### Session 2026-03-07

- Q: 차단 화면에서 뒤로가기 버튼을 누르면 어떻게 되는가? → A: 차단 사이트가 아닌 직전의 안전한 페이지로 이동 (Option A)
- Q: 집중 경고 메시지 방식은 무엇인가? → A: 고정된 단일 메시지 (항상 같은 집중 유도 문구 표시) (Option A)

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 집중 경고 차단 화면 표시 (Priority: P1)

사용자가 할일을 "진행 중" 상태로 설정한 후 차단된 사이트에 접속하면, 기존의 단순한 차단 안내 대신 **현재 업무에 집중하라는 경고 문구가 포함된 전체 화면**이 표시된다. 이 화면은 사용자에게 현재 진행 중인 할일 제목을 보여주고, 집중을 유도하는 동기 부여 메시지를 전달한다.

**Why this priority**: 이 기능의 핵심 가치이다. 단순 차단 안내에서 "집중 유도 경고 화면"으로의 전환이 사용자 경험의 근본적 변화를 만든다. 이 스토리만으로도 사용자에게 실질적인 집중 효과를 제공할 수 있다.

**Independent Test**: 할일을 진행 상태로 변경한 후 차단된 사이트에 접속하여, 집중 경고 화면이 올바르게 표시되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 사용자가 할일을 "진행 중"으로 설정하고 차단 사이트 목록에 "youtube.com"이 등록되어 있을 때, **When** 사용자가 youtube.com에 접속하면, **Then** 집중 경고 화면이 전체 브라우저 탭에 표시되고, 현재 진행 중인 할일 제목이 화면에 보인다.
2. **Given** 집중 경고 화면이 표시된 상태에서, **When** 사용자가 화면을 확인하면, **Then** 집중을 유도하는 경고 메시지(예: "지금은 업무에 집중할 시간입니다")가 눈에 띄게 표시되어 있다.
3. **Given** 집중 경고 화면이 표시된 상태에서, **When** 사용자가 화면을 확인하면, **Then** 차단 해제 방법에 대한 안내(팝업에서 할일 상태 변경)가 포함되어 있다.
4. **Given** 할일 제목 정보가 전달되지 않은 경우, **When** 차단 화면이 표시되면, **Then** 할일 제목 대신 기본 집중 메시지만 표시되고, 화면이 정상적으로 렌더링된다.

---

### User Story 2 - 시각적으로 몰입감 있는 차단 화면 디자인 (Priority: P2)

차단 화면이 단순한 텍스트 안내가 아닌, 시각적으로 임팩트 있고 몰입감을 주는 디자인으로 구성된다. 경고 아이콘, 색상 대비, 타이포그래피 강조 등을 통해 사용자가 즉시 "지금은 집중해야 할 시간"이라는 것을 인식할 수 있어야 한다.

**Why this priority**: 경고 메시지의 내용만큼이나 시각적 전달력이 중요하다. 잘 디자인된 차단 화면은 사용자의 습관적 사이트 접속을 효과적으로 끊어주는 심리적 장벽 역할을 한다.

**Independent Test**: 차단 화면을 띄웠을 때 시각적 요소들(경고 아이콘, 색상, 레이아웃)이 의도된 대로 표시되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 차단 화면이 표시될 때, **When** 사용자가 화면을 보면, **Then** 집중을 상징하는 시각적 아이콘이 화면 상단에 크게 표시된다.
2. **Given** 차단 화면이 표시될 때, **When** 사용자가 화면을 보면, **Then** 경고 메시지가 강조된 색상과 큰 글씨로 표시되어 즉시 눈에 들어온다.
3. **Given** 차단 화면이 표시될 때, **When** 사용자가 화면을 보면, **Then** 현재 진행 중인 할일 제목이 구분되는 스타일로 강조 표시된다.
4. **Given** 차단 화면이 다양한 화면 크기(데스크톱, 태블릿 해상도)에서 표시될 때, **When** 사용자가 확인하면, **Then** 화면 요소들이 가려지거나 깨지지 않고 정상적으로 표시된다.

---

### User Story 3 - 차단 화면에 진행 중인 할일 제목 실시간 연동 (Priority: P3)

차단 화면이 표시될 때, 현재 "진행 중"인 할일의 제목이 정확하게 전달되어 화면에 표시된다. 사용자가 어떤 업무에 집중해야 하는지 구체적으로 알 수 있다.

**Why this priority**: 할일 제목이 표시되어야 사용자가 "무엇을 해야 하는지"를 명확히 인지할 수 있다. P1의 기본 경고 메시지에 구체성을 더해주는 기능이다.

**Independent Test**: 특정 할일을 진행 중으로 설정한 후 차단 사이트에 접속하여, 해당 할일의 제목이 차단 화면에 정확히 표시되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 사용자가 "보고서 작성" 할일을 진행 중으로 설정했을 때, **When** 차단 사이트에 접속하면, **Then** 차단 화면에 "보고서 작성"이라는 할일 제목이 표시된다.
2. **Given** 할일 제목이 매우 긴 경우(50자 이상), **When** 차단 화면이 표시되면, **Then** 제목이 적절히 줄 바꿈 되거나 말줄임 처리되어 화면 레이아웃이 깨지지 않는다.
3. **Given** 사용자가 진행 중인 할일을 변경(다른 할일로 전환)한 후, **When** 차단 사이트에 다시 접속하면, **Then** 새로 진행 중인 할일의 제목이 차단 화면에 표시된다.

---

### Edge Cases

- 할일 제목에 특수 문자(이모지, HTML 엔티티 등)가 포함된 경우 차단 화면에서 올바르게 렌더링되는가?
- 브라우저의 다크 모드/라이트 모드 시스템 설정에 관계없이 차단 화면이 의도된 대로 표시되는가?
- 차단 화면이 표시된 상태에서 사용자가 뒤로가기 버튼을 누르면, 차단 사이트가 아닌 직전 안전한 페이지로 이동한다.
- 동시에 여러 탭에서 차단 사이트에 접속한 경우, 모든 탭에서 차단 화면이 올바르게 표시되는가?
- 확장 프로그램이 업데이트되거나 재시작된 직후 차단 화면이 정상 동작하는가?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 차단 화면은 사용자에게 현재 업무에 집중하라는 경고 메시지를 표시해야 한다 (MUST). 메시지는 단순 차단 안내가 아닌, 집중을 유도하는 고정된 단일 동기 부여 문구여야 한다. 메시지 내용은 매 접속마다 동일하게 표시된다.
- **FR-002**: 차단 화면은 현재 "진행 중"인 할일의 제목을 표시해야 한다 (MUST). 할일 제목이 없는 경우에도 기본 집중 경고 메시지는 표시되어야 한다.
- **FR-003**: 차단 화면은 시각적으로 임팩트 있는 디자인이어야 한다 (MUST). 경고 아이콘, 강조 색상, 적절한 타이포그래피를 사용하여 사용자가 "집중 시간"임을 즉시 인식할 수 있어야 한다.
- **FR-004**: 차단 화면은 차단 해제 방법을 안내해야 한다 (MUST). 사용자가 팝업에서 할일 상태를 변경하면 차단이 해제된다는 정보를 포함해야 한다.
- **FR-005**: 차단 화면은 전체 브라우저 탭 영역을 차지해야 한다 (MUST). 원래 접속하려던 사이트의 콘텐츠가 전혀 보이지 않아야 한다.
- **FR-006**: 진행 중인 할일의 제목이 차단 화면으로 정확하게 전달되어야 한다 (MUST). 현재 활성화된 할일 정보가 차단 화면에서 접근 가능해야 한다.
- **FR-007**: 할일 제목이 긴 경우(50자 이상) 화면 레이아웃이 깨지지 않도록 적절히 처리해야 한다 (MUST).
- **FR-008**: 차단 화면은 일반적인 데스크톱 브라우저 해상도(1024px 이상)에서 정상적으로 표시되어야 한다 (MUST).
- **FR-009**: 사용자가 차단 화면에서 브라우저 뒤로가기를 실행하면, 차단 사이트가 아닌 직전의 안전한 페이지로 이동해야 한다 (MUST). 차단 화면이 브라우저 히스토리 스택에 잔류하여 반복 표시되지 않아야 한다.

### Key Entities

- **집중 경고 화면 (Focus Block Screen)**: 차단된 사이트 접속 시 표시되는 전체 화면 페이지. 집중 경고 메시지, 진행 중인 할일 제목, 차단 해제 안내를 포함한다.
- **진행 중인 할일 (Active Task)**: 현재 "진행 중(in_progress)" 상태인 할일. 차단 화면에서 제목이 표시되는 대상이다. 기존 시스템에서 관리되는 엔티티를 활용한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 차단 사이트 접속 시 1초 이내에 집중 경고 화면이 표시된다.
- **SC-002**: 차단 화면에서 사용자의 80% 이상이 현재 진행 중인 할일이 무엇인지 즉시 인식할 수 있다 (할일 제목의 시각적 강조를 통해).
- **SC-003**: 차단 화면을 본 사용자의 70% 이상이 추가 행동(뒤로가기 등) 없이 업무로 복귀한다.
- **SC-004**: 모든 일반 데스크톱 해상도(1024px~2560px)에서 차단 화면의 레이아웃이 깨지지 않고 정상 표시된다.
- **SC-005**: 할일 제목이 50자를 초과하는 경우에도 화면이 정상적으로 표시되고 제목이 읽을 수 있는 형태로 처리된다.

## Assumptions

- 기존 사이트 차단 기능은 유지하되, 차단 시 표시되는 화면의 내용과 디자인을 개선하는 것이 이 기능의 범위이다.
- 현재 이미 차단 화면이 존재하므로, 이를 확장하여 집중 경고 화면으로 업그레이드한다.
- 진행 중인 할일 제목이 차단 화면에 전달되어 표시될 수 있어야 한다.
- 사용자 커스텀 화면 업로드 기능은 이 스펙의 범위에 포함하지 않는다. 사용자가 언급한 "추후 사용자가 커스텀해서 업로드하여 구성" 기능은 별도 기능으로 분리하여 추후 기획한다.
- 다크 모드 지원은 이 버전의 범위에 포함하지 않으며, 차단 화면은 단일 테마(라이트 계열)로 제공한다.
