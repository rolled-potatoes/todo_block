# Feature Specification: Todoist 집중 차단기

**Feature Branch**: `001-todoist-focus-blocker`
**Created**: 2026-03-06
**Status**: Draft
**Input**: User description: "todoist의 오늘 todo를 조회하고 이를 중단/진행/완료 처리할 수 있다. 진행 중일때는 특정 홈페이지 접속을 제한하여 해당 아이템에 집중할 수 있도록한다. 제한할 홈페이지는 내가 접속한 페이지에서 append 하거나 이미 있는 항목은 제거할 수 있다."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Todoist 오늘 할일 조회 (Priority: P1)

사용자가 크롬 익스텐션 팝업을 열면, Todoist에 등록된 오늘 날짜의 할일 목록이 표시된다.
각 할일은 제목과 현재 상태(중단/진행/완료)를 함께 보여준다.

**Why this priority**: 모든 기능의 기반이 되는 핵심 기능. Todoist 데이터를 조회할 수 없으면
나머지 기능이 동작할 수 없다.

**Independent Test**: Todoist 계정을 연결한 뒤 팝업을 열어 오늘의 할일이 올바르게
표시되는지 확인하여 독립적으로 테스트 가능.

**Acceptance Scenarios**:

1. **Given** 사용자가 Todoist API 토큰을 설정한 상태, **When** 팝업을 열면, **Then** 오늘 마감인 할일 목록이 제목과 함께 표시된다
2. **Given** Todoist에 오늘 할일이 없는 상태, **When** 팝업을 열면, **Then** "오늘 할일이 없습니다" 안내 메시지가 표시된다
3. **Given** API 토큰이 설정되지 않은 상태, **When** 팝업을 열면, **Then** Todoist 연동 설정 안내가 표시된다
4. **Given** 네트워크 연결이 불안정한 상태, **When** 팝업을 열면, **Then** 적절한 오류 메시지와 재시도 옵션이 표시된다

---

### User Story 2 - 할일 상태 관리 및 사이트 차단 (Priority: P2)

사용자가 특정 할일을 "진행" 상태로 변경하면, 미리 등록한 차단 목록의 사이트 접속이 즉시
제한된다. "중단" 또는 "완료"로 변경하면 차단이 해제된다. 동시에 하나의 할일만 "진행"
상태일 수 있다.

**Why this priority**: 프로젝트의 핵심 가치인 "집중을 위한 사이트 차단" 기능.
할일 조회 다음으로 중요하며, 이 기능이 없으면 단순한 할일 뷰어에 불과하다.

**Independent Test**: 할일을 "진행"으로 전환 후 차단 목록의 사이트에 접속을 시도하여
차단 여부를 확인.

**Acceptance Scenarios**:

1. **Given** 차단 목록에 사이트가 등록된 상태, **When** 할일을 "진행"으로 변경하면, **Then** 해당 사이트 접속 시 차단 안내 페이지가 표시된다
2. **Given** 할일이 "진행" 상태, **When** "중단"으로 변경하면, **Then** 차단이 즉시 해제되고 사이트에 정상 접속된다
3. **Given** 할일이 "진행" 상태, **When** "완료"로 변경하면, **Then** 차단이 해제되고 Todoist에서도 완료 처리된다
4. **Given** 할일 A가 "진행" 상태, **When** 할일 B를 "진행"으로 변경하면, **Then** 할일 A는 자동으로 "중단"이 되고 할일 B가 "진행"이 된다

---

### User Story 3 - 차단 사이트 목록 관리 (Priority: P3)

사용자가 현재 브라우저에서 접속 중인 페이지를 차단 목록에 추가하거나, 이미 등록된 항목을
제거할 수 있다. 차단 목록은 브라우저를 닫아도 유지된다.

**Why this priority**: 차단 기능을 활용하려면 차단할 사이트를 관리할 수 있어야 한다.
US2가 구현된 이후에 의미 있게 사용 가능하다.

**Independent Test**: 임의의 사이트에 접속한 상태에서 차단 목록 추가 버튼을 눌러
등록되는지, 제거 버튼으로 삭제되는지 확인.

**Acceptance Scenarios**:

1. **Given** 특정 웹사이트에 접속한 상태, **When** 팝업에서 "현재 페이지 차단 추가" 버튼을 누르면, **Then** 해당 사이트의 도메인이 차단 목록에 추가된다
2. **Given** 차단 목록에 사이트가 있는 상태, **When** 해당 항목 옆 제거 버튼을 누르면, **Then** 차단 목록에서 즉시 제거된다
3. **Given** 차단 목록에 사이트를 추가한 뒤 브라우저를 재시작, **When** 팝업을 다시 열면, **Then** 이전에 추가한 차단 목록이 그대로 표시된다
4. **Given** 이미 차단 목록에 있는 사이트에 접속한 상태, **When** 팝업을 열면, **Then** "현재 페이지 차단 추가" 대신 "차단 해제" 버튼이 표시된다

---

### Edge Cases

- Todoist API 토큰이 만료되거나 유효하지 않은 경우 어떻게 처리하는가?
- 진행 중인 할일이 있는 상태에서 브라우저를 완전히 종료하고 재시작하면 차단 상태가 유지되는가?
- 차단된 사이트의 서브도메인(예: mail.google.com vs calendar.google.com)도 차단되는가?
- Todoist에서 직접 할일을 삭제한 경우 익스텐션에서 어떻게 반영되는가?
- 차단 목록이 비어있는 상태에서 할일을 "진행"으로 변경하면 어떻게 되는가?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 Todoist에서 오늘 날짜의 할일 목록을 조회할 수 있어야 한다
- **FR-002**: 사용자는 각 할일의 상태를 "중단", "진행", "완료" 중 하나로 변경할 수 있어야 한다
- **FR-003**: 동시에 최대 하나의 할일만 "진행" 상태일 수 있어야 한다
- **FR-004**: 할일이 "진행" 상태가 되면 차단 목록의 사이트 접속이 즉시 제한되어야 한다
- **FR-005**: 할일이 "중단" 또는 "완료" 상태가 되면 사이트 차단이 즉시 해제되어야 한다
- **FR-006**: 할일을 "완료"로 변경하면 Todoist에서도 해당 할일이 완료 처리되어야 한다
- **FR-007**: 사용자는 현재 접속 중인 웹페이지의 도메인을 차단 목록에 추가할 수 있어야 한다
- **FR-008**: 사용자는 차단 목록에서 기존 항목을 제거할 수 있어야 한다
- **FR-009**: 차단 목록은 브라우저 세션 간 영속적으로 유지되어야 한다
- **FR-010**: 차단된 사이트 접속 시 차단 안내 페이지가 표시되어야 한다
- **FR-011**: 시스템은 API 토큰 직접 입력 방식으로 Todoist 사용자 인증을 지원해야 한다 (1단계, 향후 OAuth2 확장 가능 구조)
- **FR-012**: API 토큰이 유효하지 않은 경우 사용자에게 명확한 오류 메시지와 재설정 안내를 제공해야 한다

### Key Entities

- **할일 항목 (Todo Item)**: Todoist에서 가져온 오늘의 할일. 제목, 상태(중단/진행/완료), Todoist 고유 식별자, 마감일을 포함.
- **차단 사이트 (Blocked Site)**: 사용자가 등록한 차단 대상 웹사이트. 도메인 패턴, 추가 일시를 포함. 진행 중인 할일이 있을 때만 차단 활성화.
- **사용자 설정 (User Settings)**: Todoist API 토큰, 차단 목록, 사용자 환경설정 등을 포함하는 사용자 구성 정보.

### Assumptions

- 차단은 도메인 단위로 동작한다 (서브도메인 포함). 예: "youtube.com"을 차단하면 "www.youtube.com", "m.youtube.com" 모두 차단.
- 1단계에서는 API 토큰 직접 입력 방식을 사용하며, 향후 OAuth2 인증을 추가할 수 있는 구조로 설계한다.
- 차단 안내 페이지는 현재 진행 중인 할일 제목과 "돌아가기" 안내를 포함한다.
- 할일 목록은 팝업을 열 때마다 Todoist에서 최신 데이터를 가져온다.
- "중단" 상태는 익스텐션 내부에서만 관리되며 Todoist에는 반영하지 않는다. "완료"만 Todoist에 동기화한다.
- 차단 목록이 비어있는 상태에서 할일을 "진행"으로 변경하면 상태만 전환되고 차단은 미동작한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 팝업을 열면 3초 이내에 오늘의 할일 목록이 표시된다
- **SC-002**: 할일 상태를 "진행"으로 변경하면 1초 이내에 사이트 차단이 활성화된다
- **SC-003**: 차단된 사이트 접속 시 100% 확률로 접근이 차단되고 안내 페이지가 표시된다
- **SC-004**: 사용자가 현재 페이지를 차단 목록에 추가하는 동작을 2번의 클릭 이내로 완료할 수 있다
- **SC-005**: 처음 사용하는 사용자가 2분 이내에 API 토큰을 설정하고 첫 할일을 진행 상태로 전환할 수 있다
- **SC-006**: 진행 중인 할일이 없는 상태에서는 모든 사이트에 정상적으로 접속할 수 있다
